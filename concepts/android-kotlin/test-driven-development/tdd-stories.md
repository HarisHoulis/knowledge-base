# Kotlin TDD - New Project and First Stories

In this video, Duncan demonstrates applying Extreme Programming (XP) practices to restart the Gilded Rose project from scratch. He emphasizes using customer stories to drive development, starting with simple stories like adding items to stock and printing the stock list. The approach follows TDD with test-first coding, writing only enough code to pass the failing test, and refactoring. He also practices 'sitting on your cards'—focusing only on the current story and deferring future ones. The implementation uses in-memory state and the IDE's code generation features to keep the design minimal and incremental.

- Use customer stories to define and prioritize features, starting with the simplest ones.
- Apply TDD strictly: write a failing test before writing any production code, then make it pass and refactor.
- Practice 'sit on your cards' to avoid premature design decisions and focus on the current story.
- Use in-memory storage for early iterations to avoid database complexity.
- Leverage IDE features like automatic code generation to speed up TDD cycles.
