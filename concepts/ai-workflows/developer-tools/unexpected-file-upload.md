# Grok’s CLI caught uploading all local files to the cloud

The Grok CLI, a developer tool for interacting with the Grok 4.5 coding model, was discovered to be uploading all local files to the cloud without explicit user consent. This behavior was flagged by developers who noticed unusual network activity, leading to backlash and concerns about data privacy. The incident underscores the need for transparency in AI-powered tools that access local filesystems, as well as the risks of unintended data exfiltration.

- The Grok CLI automatically uploaded all local files to the cloud, likely unintentionally.
- Developers discovered the behavior and raised privacy concerns publicly.
- The incident highlights the importance of clear data handling policies in developer tools.
- Trust in AI coding assistants can be eroded by such privacy breaches.
