# Kill your MEMORY.md

Matt Pocock argues that agent memory, such as the MEMORY.md file in Claude Code, often does more harm than good. He recommends disabling auto-memory and keeping AI agents completely stateless to ensure predictable behavior. By avoiding persistent memory, developers can prevent the agent from making decisions based on outdated or irrelevant context, leading to more reliable and consistent outputs.

- Agent memory can introduce unpredictability by causing the AI to rely on stale or incorrect information.
- Disabling auto-memory features, like MEMORY.md in Claude Code, helps maintain statelessness.
- A stateless AI agent is more predictable and easier to debug because each interaction starts fresh.
- Developers should carefully consider whether persistent memory is truly necessary for their use case.
